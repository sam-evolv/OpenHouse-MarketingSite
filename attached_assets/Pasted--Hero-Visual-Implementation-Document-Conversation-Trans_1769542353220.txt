# Hero Visual Implementation: Document → Conversation Transformation

## Design Philosophy

This visual must feel **premium, understated, and intelligent**. Think Apple product reveals, not generic SaaS. Every element earns its place. Animation is subtle and purposeful, never flashy or gimmicky.

**Key principles:**
- Depth through layering and shadows, not bright colours
- Gold used as an accent, never overwhelming
- Smooth, slow animations that feel considered
- Glass/frosted effects for premium material feel
- Generous negative space

## Element 2: Particle Flow (Center)

**Concept:** Golden particles that appear to emanate from the documents and flow toward the chat interface. NOT a chaotic spray - a controlled, elegant stream suggesting data transformation.

### Particle Design

Each particle is a small, soft circle with blur and varying opacity:

```tsx
// Single particle
className="absolute rounded-full bg-amber-400"
style={{
  width: '6px',
  height: '6px',
  filter: 'blur(1px)',
  boxShadow: '0 0 8px rgba(212, 175, 55, 0.5)',
}}
```

### Particle Path

Particles should follow a subtle curved path (bezier) from left to right, not a straight line:

```
Start (documents) ──╮
                     ╰──────────────────╮
                                         ╰── End (chat)
```

### Particle Animation

```css
@keyframes particle-flow {
  0% {
    transform: translateX(0) translateY(0) scale(0);
    opacity: 0;
  }
  10% {
    transform: translateX(20px) translateY(-5px) scale(1);
    opacity: 0.8;
  }
  50% {
    transform: translateX(150px) translateY(-15px) scale(0.8);
    opacity: 1;
  }
  90% {
    transform: translateX(280px) translateY(-5px) scale(0.6);
    opacity: 0.6;
  }
  100% {
    transform: translateX(300px) translateY(0) scale(0);
    opacity: 0;
  }
}
```

### Particle Generation

Generate 12-15 particles, each with:
- Staggered animation delay (spread over 3 seconds)
- Slightly different vertical offset
- Varying sizes (4px to 8px)
- Varying opacity (0.4 to 1.0)

```tsx
const particles = Array.from({ length: 15 }, (_, i) => ({
  id: i,
  delay: (i * 0.2) % 3, // Stagger over 3 seconds
  yOffset: Math.sin(i * 0.5) * 20, // Slight wave pattern
  size: 4 + Math.random() * 4,
  opacity: 0.4 + Math.random() * 0.6,
}));

// Render
{particles.map((p) => (
  <div
    key={p.id}
    className="absolute rounded-full bg-amber-400"
    style={{
      width: `${p.size}px`,
      height: `${p.size}px`,
      left: '140px', // Start from document edge
      top: `calc(50% + ${p.yOffset}px)`,
      filter: 'blur(1px)',
      boxShadow: '0 0 8px rgba(212, 175, 55, 0.4)',
      animation: `particle-flow 3s ease-in-out infinite`,
      animationDelay: `${p.delay}s`,
      opacity: p.opacity,
    }}
  />
))}
```

### Subtle Light Trail

Behind the particles, add a very subtle gradient beam:

```tsx
// Light trail (positioned behind particles)
<div 
  className="absolute h-[2px] opacity-20"
  style={{
    left: '160px',
    right: '160px',
    top: '50%',
    background: 'linear-gradient(90deg, transparent, rgba(212, 175, 55, 0.3) 20%, rgba(212, 175, 55, 0.3) 80%, transparent)',
  }}
/>
```

## Responsive Behaviour

### Desktop (1024px+)
- Full layout as described
- Documents: 120x160px
- Chat: 280px wide
- Gap between elements: 64px (gap-16)

### Tablet (768-1023px)
- Reduce gap to 32px (gap-8)
- Documents: 100x140px
- Chat: 260px wide
- Particles: Reduce count to 10

### Mobile (<768px)
- Stack vertically or hide entirely
- Consider showing only the chat interface on mobile
- Or simplify to just the particle trail as a divider

```tsx
// Mobile alternative - simplified
<div className="md:hidden">
  {/* Just show a subtle gold line with a few particles */}
  <div className="h-px w-48 mx-auto bg-gradient-to-r from-transparent via-amber-400/50 to-transparent" />
</div>
```

Before shipping, verify:

- [ ] Animations are smooth (60fps)
- [ ] Particles don't cause layout shift
- [ ] Chat glow is subtle, not overwhelming
- [ ] Document stack has proper depth (shadows, layering)
- [ ] Gold colour matches brand (#D4AF37)
- [ ] Respects `prefers-reduced-motion` (disable animations)
- [ ] Works on Safari (check animation compatibility)
- [ ] Dark backgrounds match existing site (#0b0c0f, #1e222a)

## Implementation Notes for Replit

1. **Create as a separate component** - `HeroVisual.tsx` in your components folder
2. **Import Lucide icon** - `FileText` for the document icon
3. **Add keyframes to global CSS** or tailwind.config.js
4. **Test on multiple browsers** - Safari can be finicky with CSS animations
5. **Performance** - Use `will-change: transform` on animated elements if needed

This spec should give Replit everything it needs to build a premium, polished hero visual that matches your brand quality.
